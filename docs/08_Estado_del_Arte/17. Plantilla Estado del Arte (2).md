## Estado del arte que hace el autor(1 página)

En la introducción del documento se evidencia que los autores parten de la creciente integración de la tecnología en la vida diaria, especialmente en el ámbito de la salud y el entrenamiento físico. Se reconoce que actualmente existen diversas aplicaciones móviles enfocadas en el ejercicio físico, las cuales ofrecen rutinas generales, seguimiento de progreso y en algunos casos personalización básica.
Sin embargo, los autores identifican que muchas de estas soluciones no consideran de manera integral variables importantes como las limitaciones físicas del usuario, el equipo disponible o el contexto específico (por ejemplo, gimnasio o casa). Además, varias aplicaciones no ofrecen una adaptación continua de las rutinas conforme el usuario progresa.
También se menciona implícitamente que otras soluciones no integran funcionalidades adicionales como la visualización de mapas del gimnasio o la optimización del uso de equipos disponibles. Esto indica que el estado actual de las aplicaciones presenta limitaciones en cuanto a personalización profunda, interacción con el entorno físico y adaptación dinámica.
En base a esto, los autores plantean que existe una necesidad de desarrollar una solución más completa que combine personalización, seguimiento, adaptación y apoyo visual para mejorar la experiencia del usuario.

## Motivación del autor (críticas del autor a otros trabajos)(0.5 página)

La motivación principal de los autores surge debido a las limitaciones encontradas en aplicaciones existentes de entrenamiento físico. Aunque existen herramientas digitales, estas no logran ofrecer una experiencia totalmente personalizada ni adaptativa.
Los autores justifican su propuesta señalando que muchas aplicaciones no consideran factores clave como:
Limitaciones físicas del usuario
Objetivos específicos
Disponibilidad de equipos
Evolución progresiva del rendimiento
Además, se evidencia un vacío en la integración entre el entorno físico del gimnasio y la aplicación, ya que pocas soluciones permiten ubicar equipos o interactuar con el espacio real.
Por ello, los autores consideran necesario desarrollar una aplicación que no solo genere rutinas, sino que también se adapte dinámicamente al usuario y optimice su experiencia dentro del gimnasio.

## Descripción del aporte del autor(2 páginas)

El principal aporte del artículo es el desarrollo de la aplicación móvil SmartCoach, la cual funciona como un entrenador personal inteligente.
El problema que se aborda es la falta de personalización y seguimiento adecuado en los sistemas actuales de entrenamiento físico, lo cual puede afectar el rendimiento del usuario, su motivación e incluso su salud.
El impacto de este problema se refleja en:
Rutinas poco efectivas
Riesgo de lesiones
Falta de motivación
Uso ineficiente de equipos en gimnasios
Para solucionar esto, los autores proponen una aplicación que:
Genera rutinas personalizadas considerando:
Objetivos del usuario
Limitaciones físicas
Tiempo disponible
Equipos disponibles
Adapta las rutinas automáticamente según el progreso del usuario.
Permite el seguimiento del rendimiento mediante registro de datos.
Proporciona guías visuales (imágenes y descripciones) para ejecutar correctamente los ejercicios.
Integra un mapa del gimnasio que permite ubicar equipos.
Facilita la interacción entre usuarios y administradores del gimnasio.
Este aporte no solo mejora la experiencia del usuario, sino que también contribuye a una mejor organización de los recursos del gimnasio y a la reducción de riesgos durante el entrenamiento.

## Proceso para obtener el aporte que considera el autor(1. página)

Para desarrollar la solución, los autores utilizan un enfoque basado en ingeniería de requerimientos.
El proceso incluye:
Sesiones colaborativas para identificar necesidades
Análisis de usuarios (clientes y administradores)
Definición de requerimientos funcionales y no funcionales
Priorización de requerimientos según:
Valor para el usuario
Impacto en el negocio
Viabilidad técnica
Además, se emplea un modelo arquitectónico cliente-servidor de 2 capas, donde:
El cliente es la aplicación móvil
El servidor maneja la lógica de negocio y base de datos
Tecnologías utilizadas:
Android Studio (Java)
PostgreSQL
API REST con Spring Boot
JWT para autenticación

## Proceso para resolver el problema considerado por el autor(1. página)

El problema se resuelve mediante la implementación de un sistema que integra múltiples funcionalidades.
Primero, se recopila información del usuario (objetivos, estado físico, equipos disponibles). Luego, el sistema genera una rutina personalizada.
Posteriormente:
Se registra el progreso del usuario
Se adapta la rutina automáticamente
Se brinda retroalimentación continua
Además, el sistema incluye:
Mapas del gimnasio
Registro de equipos
Gestión de usuarios
Respecto a pruebas, el documento menciona:
Validación de requerimientos con el director del proyecto
Verificación mediante casos de uso
Relación con casos de prueba
Esto asegura que el sistema cumpla con los requerimientos definidos.

## Métricas que el autor usa y resultado que obtiene. Comentar (los resultados son mejores respecto a otros)(0.5 página)

El documento no presenta métricas cuantitativas como porcentaje de mejora o precisión, lo cual es una limitación.
Sin embargo, se consideran métricas cualitativas como:
Usabilidad
Rendimiento
Seguridad
Fiabilidad
Disponibilidad
Estas métricas están asociadas a los requerimientos no funcionales del sistema.
En general, se espera que la solución mejore la experiencia del usuario respecto a otras aplicaciones, aunque no se presentan resultados comparativos numéricos.
Se considera que el uso de métricas cualitativas es adecuado para una fase de diseño, pero sería recomendable incluir métricas cuantitativas como:
Nivel de satisfacción del usuario
Reducción de errores en ejercicios
Mejora en el rendimiento físico
Esto permitiría validar mejor la efectividad del sistema.

## Observaciones y/o críticas suyas al artículo(0.5 página)

Una de las principales críticas es que el documento se enfoca más en la especificación de requerimientos que en la validación experimental del sistema.
No se presentan:
Pruebas reales con usuarios
Resultados medibles
Comparaciones con otras aplicaciones
Además, algunas funcionalidades importantes como integración con wearables no están incluidas.
Como aporte, se podría mejorar la propuesta incorporando:
Inteligencia artificial más avanzada para predicción de rendimiento
Integración con dispositivos wearables
Uso de visión artificial para corrección de postura
Métricas cuantitativas de evaluación
Pruebas piloto con usuarios reales
