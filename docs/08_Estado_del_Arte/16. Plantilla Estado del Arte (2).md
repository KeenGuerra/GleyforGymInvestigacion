## Estado del arte que hace el autor(1 página)

El artículo presenta un análisis de diferentes enfoques utilizados previamente para el cálculo del gasto energético (Energy Expenditure - EE) y el monitoreo de actividad física. En primer lugar, los autores mencionan métodos tradicionales como el uso de acelerómetros combinados con modelos estadísticos, específicamente ecuaciones de regresión lineal, para estimar las calorías consumidas durante actividades físicas .
Además, se mencionan métodos más precisos desde el punto de vista médico, como el método de Doubly Labeled Water (DLW) y la calorimetría indirecta, los cuales permiten calcular el gasto energético mediante procesos fisiológicos como el análisis de dióxido de carbono o el intercambio de oxígeno. Sin embargo, estos métodos son invasivos, costosos y poco prácticos para el uso diario .
También se analizan aplicaciones móviles existentes como Accupedo y Pacer, las cuales permiten registrar actividades físicas y calcular calorías. Sin embargo, estas presentan limitaciones importantes, ya que requieren que el usuario ingrese manualmente las actividades, lo cual genera esfuerzo adicional y posibles errores. Además, muchas de estas aplicaciones utilizan valores predefinidos para calcular el gasto energético, lo que reduce la precisión .
Por otro lado, se menciona el uso de técnicas de machine learning en investigaciones previas para el reconocimiento de actividades físicas. Sin embargo, dichas investigaciones no integran completamente el cálculo del gasto energético junto con el reconocimiento automático de actividades .
En base a esto, el artículo identifica una necesidad clara de desarrollar soluciones más precisas, automáticas y accesibles, que aprovechen los sensores de los smartphones sin requerir intervención constante del usuario.

## Motivación del autor (críticas del autor a otros trabajos)(0.5 página)

Los autores critican principalmente la falta de precisión y automatización en los métodos actuales para calcular el gasto energético. Señalan que muchas aplicaciones dependen de estimaciones generales como el uso de multiplicadores de actividad basados en percepciones subjetivas del usuario, lo cual puede generar resultados poco confiables .
Asimismo, critican que los sistemas existentes requieren ingreso manual de datos, lo que resulta tedioso y poco práctico para los usuarios. También resaltan que los dispositivos wearables pueden presentar errores en la medición de actividad física.
La motivación principal del estudio surge de la necesidad de mejorar la precisión del cálculo del gasto energético y eliminar la intervención manual del usuario, utilizando tecnologías accesibles como sensores de smartphones y algoritmos de machine learning.

## Descripción del aporte del autor(2 páginas)

El aporte principal del artículo es el desarrollo de una aplicación móvil llamada CalAid, la cual permite monitorear la actividad física del usuario y calcular el gasto energético de manera automática.
El problema que aborda el estudio es el sedentarismo creciente en la población y la falta de herramientas precisas y prácticas para monitorear la actividad física diaria. Este problema tiene impacto tanto en la salud de las personas como en la calidad de vida, ya que la falta de actividad física está relacionada con enfermedades crónicas.
La solución propuesta consiste en una aplicación que utiliza sensores del smartphone (acelerómetro, giroscopio y contador de pasos) para detectar actividades físicas en tiempo real. A diferencia de otras aplicaciones, esta no requiere que el usuario registre manualmente sus actividades.
El sistema implementa un algoritmo de machine learning para reconocer actividades como caminar, subir escaleras, estar sentado o acostado. Luego, utiliza los valores de MET (Metabolic Equivalent of Task) para calcular el gasto energético en calorías.
Además, el sistema calcula el TDEE (Total Daily Energy Expenditure) de manera más precisa, basándose en el historial real de actividades del usuario, en lugar de usar valores predefinidos.
Este aporte es importante porque mejora la precisión de las aplicaciones fitness, reduce el esfuerzo del usuario y promueve un estilo de vida más activo.

## Proceso para obtener el aporte que considera el autor(1. página)

Para desarrollar la solución, los autores utilizaron una combinación de tecnologías:
Sensores del smartphone (acelerómetro, giroscopio y contador de pasos)
Dataset UCI HAR (Human Activity Recognition)
Algoritmos de machine learning supervisado
El proceso consiste en:
Recolección de datos de sensores
Filtrado de ruido mediante filtros (mediana y Butterworth)
Extracción de características (81 features)
Entrenamiento del modelo con dataset UCI HAR
Clasificación de actividades
Cálculo del gasto energético usando MET
También se utiliza una arquitectura cliente-servidor con base de datos NoSQL y procesamiento en la nube (Azure) .

## Proceso para resolver el problema considerado por el autor(1. página)

El problema se resuelve mediante un modelo de clasificación supervisada utilizando Linear Support Vector Classifier (LinearSVC).
🔹 Métricas usadas:
Accuracy: 83.5%
Precisión promedio: 88%
Recall promedio: 81%
F1-score: 81%
El modelo presenta dificultades en algunas clases como “walking upstairs” (49% recall), lo que indica que aún hay margen de mejora .
Los autores realizaron pruebas con datos reales obtenidos de usuarios que realizaron diferentes actividades físicas. Además, se evaluó la experiencia del usuario, obteniendo resultados positivos en cuanto a facilidad de uso.
🔹 Comentario:
Las métricas son aceptables para un sistema de reconocimiento de actividades, pero podrían mejorar con más datos o modelos más avanzados (por ejemplo redes neuronales).

## Métricas que el autor usa y resultado que obtiene. Comentar (los resultados son mejores respecto a otros)(0.5 página)

Una de las principales limitaciones del estudio es que el sistema depende de que el usuario lleve el smartphone consigo, lo cual no siempre ocurre.
Además, el modelo tiene dificultades para reconocer algunas actividades con alta precisión. También el número de actividades detectadas es limitado.
Otra debilidad es que no se validó completamente el cálculo de calorías con métodos médicos como DLW, lo cual podría mejorar la confiabilidad.

## Observaciones y/o críticas suyas al artículo(0.5 página)

Como aporte adicional, se podría:
Implementar modelos más avanzados como Deep Learning para mejorar la precisión
Integrar sensores adicionales (smartwatch)
Aumentar la cantidad de actividades reconocidas
Validar los resultados con métodos médicos reales
Agregar recomendaciones personalizadas de ejercicio y nutrición
